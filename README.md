# tff-poc

A proof-of-concept pipeline comparing **centralized**, **federated**, and **federated + differentially private (DP)** training on synthetic patient data, built to generate the empirical results behind an MPhil thesis on epsilon selection in differential privacy as de facto technical policymaking.

Thesis linked in repo: [Tice_Parameterisation_Privacy.pdf](https://github.com/alyssajotice/tff-poc/blob/master/Tice_Parameterisation_Privacy.pdf)

The core question: as you tighten the privacy budget (lower epsilon → more noise), how much model accuracy do you give up — and at what point does that trade-off become a policy decision rather than a purely technical one?

## What it does

1. Trains a binary classifier predicting **prediabetes** from `age`, `bmi`, and `gender`, using [Synthea](https://github.com/synthetichealth/synthea)-generated synthetic patient records.
2. Runs the same task across six environments:
   - Centralized baseline
   - Federated baseline (no privacy, 5 simulated clinics)
   - Federated + DP at four privacy budgets: ε = 100, 10, 1, 0.1
3. Plots the resulting privacy–utility curve.

## Results

From `visuals.py` (final thesis figures):

| Setting | Accuracy |
|---|---|
| Centralized baseline | 73.4% |
| Federated baseline (no DP) | 68.3% |
| Federated DP, ε = 100 | 62.2% |
| Federated DP, ε = 10 | 68.6% |
| Federated DP, ε = 1 | 69.7% |
| Federated DP, ε = 0.1 | 62.1% |

ε = 1 performs best among the DP settings here — the noise calibration (not just the nominal epsilon) drives the actual accuracy/privacy trade-off, which is part of the thesis argument.

## Repo structure

| File | Purpose |
|---|---|
| `baseline_training.py` | Loads and preprocesses Synthea data (`patients.csv`, `observations.csv`, `conditions.csv`), builds the prediabetes label, trains the centralized Keras baseline, saves `baseline_model.h5` |
| `federated_baseline.py` | Partitions the same data into 5 simulated "clinics" and runs vanilla (non-private) Federated Averaging via TensorFlow Federated |
| `federated_dp.py` | Same as above, but wraps the aggregator in a Gaussian DP mechanism at a single fixed epsilon |
| `federated_episilon_sweep.py` | Runs the DP-federated process across multiple noise/epsilon settings and prints a comparison table |
| `partition.py` | Standalone helper for splitting the dataset into client partitions ("clinics") |
| `process_data.py` | Earlier/exploratory preprocessing script (diabetes label, different from the prediabetes pipeline used in the final results) |
| `data_census.py` | Data exploration / profiling script for the raw Synthea tables |
| `train_model.py` | Additional model-training experiment script |
| `mnist_federated.py` | TFF sanity-check demo on MNIST, used to validate the federated pipeline before applying it to patient data |
| `checks.ipynb` | Notebook for ad hoc validation of pipeline outputs |
| `visuals.py` | Generates the final privacy–utility curve (`privacy_utility_curve_final.png`) from the recorded results |
| `baseline_model.h5` | Saved centralized model |
| `privacy_utility_curve_final.png` | Output figure |
| `Dockerfile` | Pinned environment for reproducing the TFF/DP results |

## Setup

The TFF/JAX/TensorFlow-Privacy stack is version-sensitive, so this repo is built around a pinned Docker image rather than a loose `requirements.txt`.

```bash
docker build --platform linux/amd64 -t tff-poc .
docker run -p 8888:8888 -v $(pwd):/app tff-poc
```

This gives you:
- Python 3.10 (amd64 — required for TFF compatibility, including on Apple Silicon via emulation)
- TensorFlow 2.15.0
- TensorFlow Federated 0.65.0
- TensorFlow Privacy 0.8.12
- JAX/jaxlib 0.4.14 (pinned to avoid a known TFF 0.65.0 conflict)
- Jupyter (served on port 8888, no token)

## Running the pipeline

1. Place the Synthea-exported CSVs (`patients.csv`, `observations.csv`, `conditions.csv`) in `data/`.
2. Train the centralized baseline:
   ```bash
   python baseline_training.py
   ```
3. Run the non-private federated baseline:
   ```bash
   python federated_baseline.py
   ```
4. Run DP-federated training at a single epsilon:
   ```bash
   python federated_dp.py
   ```
5. Sweep across epsilon settings:
   ```bash
   python federated_episilon_sweep.py
   ```
6. Generate the privacy–utility plot:
   ```bash
   python visuals.py
   ```

## Data

Patient data is **synthetic**, generated by Synthea — no real patient records are used or included in this repo. Raw CSVs are expected under `data/` and are git-ignored.

## Status

This is proof-of-concept code written to support a specific thesis analysis, not a production federated learning library. Expect hardcoded paths, fixed random seeds, and minimal error handling outside of the data-loading step.
